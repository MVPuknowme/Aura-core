Quorum-based operations

SkyGrid can require read and write quorums before accepting operational state. During network partitions, quorum checks help prevent split-brain behavior, stale reads, unsafe writes, and conflicting recovery decisions.

Read quorum

Multiple nodes must agree before SkyGrid trusts status, health, route, or validator readings.

Write quorum

State changes require enough trusted confirmations before updates are promoted or acted on.

Partition safety

If the network splits, unsafe minority routes fail closed instead of forcing conflicting writes.

Recovery proof

Consensus decisions should leave logs, hashes, commits, records, or dashboard artifacts.
